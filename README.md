# 价值投资控制台

三标的定投决策辅助系统：RSI21+MA250（A股价值） + PE分位+回撤（美股科技）

| 基金 | 跟踪指数 | 数据源 | 权重 | 策略 |

| 025497 易方达价值100A | 国证价值100 (980081) | **国证指数官网**（直接指数数据） | 40% | RSI21+MA250 |

## 策略 V6 · 三标的统一规则

### 统一设定
- 定投操作日：**每月21日**（遇节假日顺延）
- 资金权重固定：40% / 40% / 20%
- 每半年（6月+12月21日）再平衡回标准权重
- 铁则：3年以上闲钱、无杠杆、每月1次、不盯盘

---

### 红利低波 / 价值100（RSI21+MA250）

**市场分区**
- 强势区：价格 ≥ MA250 → 可减仓/回补/定投
- 弱势区：价格 < MA250 → 只定投，不减仓不回补

**月度定投倍率**

| RSI21区间 | 状态 | 强势区 | 弱势区 |
|-----------|------|--------|--------|
| ≥72 | 极度过热 | 暂停 | 暂停 |
| 65-71 | 偏热 | 0.3份 | 0.3份 |
| 55-64 | 中性偏热 | 0.6份 | 0.6份 |
| 48-54 | 中性合理 | 1份 | 1份 |
| 40-47 | 偏冷低估 | 1.5份 | 1.2份 |
| <40 | 极度低估 | 2份 | 1.5份 |

**减仓**：强势区+RSI≥65+冷却外 → 卖出15%/30%，底仓≥50%
**回补**：强势区 RSI 59-63→2/3 / RSI<50→全部，30天冷却
**弱势区铁律**：不卖、不补、只投

---

### 纳斯达克100（PE分位+回撤）

**双市场分区**
- 估值分区：极度低估(<30%) / 低估(30-49%) / 合理(50-69%) / 高估(70-84%) / 极度高估(≥85%)
- 回撤分区：正常(<8%) / 中级(8-15%) / 深度(>15%)

**月度定投倍率**

| PE近7年分位 | 估值 | 基础定投 | 深度回调(>15%) |
|-------------|------|----------|----------------|
| ≥85% | 极度高估 | 暂停 | 仍暂停 |
| 70-84% | 高估 | 0.5份 | 不加仓 |
| 50-69% | 合理 | 1份 | +0.5份 |
| 30-49% | 低估 | 1.8份 | +0.5份 |
| <30% | 极度低估 | 2.5份封顶 | 维持2.5份 |

**减仓**：PE≥70%+回撤<8%+冷却外 → 卖出15%/30%，底仓≥40%
**回补**：PE 40-50%→2/3 / PE<40%→全部，60天冷却
**深度回调铁律**：回撤>15%永不减仓
**专属铁则**：5年以上闲钱、总仓位不超20%、永不止损

---

### 每月21日统一执行顺序
1. 减仓卖出 → 2. 回补买入 → 3. 月度定投

## 🚀 本地部署 & 每日自动更新

### 首次配置

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 手动运行一次，验证数据正常
python daily_update.py --no-push

# 3. 查看输出的 JSON
ls docs/
#   → data.json  (合并版，前端加载此项)
#   → 931446.json (红利低波独立数据)
#   → 980081.json (价值100独立数据)
#   → nd100.json   (纳指100独立数据)
```

### 每日自动更新（推荐）

A股指数数据通过中证/国证官网直接获取（不依赖ETF价格代理），建议在**本地 Windows 机器**上设置定时任务：

```bash
# 右键 → 以管理员身份运行 PowerShell
# 导航到项目目录，执行：
.\setup_task.ps1
```

此脚本会创建 Windows 定时任务：
- **任务名称**：RSI Monitor Daily Update
- **执行时间**：每日 **15:30**（A股收盘后）
- **执行内容**：`python daily_update.py` → 自动 git push

### 手动更新

任何时候都可以手动运行：

```bash
python daily_update.py       # 更新数据 + git push
python daily_update.py --no-push  # 只更新本地 JSON，不推送
```
