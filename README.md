## 策略 V5

### 核心理念
红利低波与价值100指数长期向上，回调是买入机会而非风险。**低位多买 > 高位少买**，通过市场分区 + RSI梯度实现差异化仓位管理。

### 市场分区
- **强势区**：价格 ≥ MA250 → 可执行全部操作（定投/减仓/回补）
- **弱势区**：价格 < MA250 → 只做定投，不减仓、不回补

### 月度定投建议（每月21日执行）

| RSI区间 | 状态 | 强势区 | 弱势区 |
|---------|------|--------|--------|
| ≥72 | 极度过热 | 暂停 | 暂停 |
| 65-71 | 偏热 | 0.3份 | 0.3份 |
| 55-64 | 中性偏热 | 0.6份 | 0.6份 |
| 48-54 | 中性合理 | 1份 | 1份 |
| 40-47 | 偏冷低估 | 1.5份 | 1.2份 |
| <40 | 极度低估 | 2份 | 1.5份 |

### 减仓规则（仅强势区）

| RSI区间 | 操作 | 说明 |
|---------|------|------|
| 65-71 | 卖出 15% | 需确认不在冷却期 |
| ≥72 | 卖出 30% | 需确认不在冷却期 |

- 底仓铁律：持仓不低于历史最高持仓的 50%

### 回补规则（仅强势区）

| 条件 | 操作 | 说明 |
|------|------|------|
| RSI 59-63 | 接回已减仓 2/3 | 触发30天冷却 |
| RSI <50 | 接回全部剩余 | 触发30天冷却 |
| 减仓后90天内RSI从未≤59 且 价格≥MA250 | 豁免接回1/3 | 不触发冷却 |

- 冷却期30天内不执行任何回补操作

### 操作日流程（每月21日）
1. 减仓 → 回补 → 定投（按顺序执行）
2. 减仓后当月不再回补（避免反复操作）
3. 冷却期内跳过回补步骤

### 弱势区铁律
- 🛡️ 不卖：弱势区永远不减仓
- 🛡️ 不补：弱势区不执行回补
- 💰 只投：弱势区按定投表正常投入

### 铁则
- 只用闲钱投资，绝不上杠杆
- 每月只操作1次，杜绝频繁交易
- 长期持有，低位多买 > 高位少买

## 技术架构

| 组件 | 方案 |
|------|------|
| 前端 | 纯静态 HTML（深色主题 · 手机优先 · 卡片式垂直流） |
| 数据引擎 | Python (AKShare + yfinance + 腾讯财经API) |
| 数据源优先级 | Yahoo Finance → 腾讯财经 → 东方财富(AKShare) → 新浪(AKShare) |
| 指标计算 | MA250（min_periods=120 支持新ETF）+ RSI21 |
| 部署 | GitHub Pages |
| 自动更新 | GitHub Actions（交易日定时执行） |

### 数据源说明
- **腾讯财经API**：国内环境最稳定，绕代理可用，前复权数据
- **Yahoo Finance**：GitHub Actions 环境首选
- **AKShare**：东方财富/新浪接口备用
- 腾讯API每次最多返回约800条，分批获取确保MA250计算数据充足

### MA250参考值
对于上市不足250个交易日的ETF（如159263），系统使用 min_periods=120 计算MA250参考值，前端标注"参考"并提示数据天数。数据自然积累至250天后自动切换为精确值。
